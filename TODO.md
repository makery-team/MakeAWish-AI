## 🛠 1단계: AI 엔진 및 AI 백엔드 (FastAPI)
*목표: 핵심 이미지 생성 및 인페인팅 로직 고도화*

- [x] **Gemini 3.1 Flash 연동**: 기본적인 이미지 인페인팅 로직 구현 완료.
- [x] **페르소나 기반 프롬프트 최적화**: 버터크림 질감 및 파스텔톤 화풍 유지 설정 완료.
- [x] **참고 이미지(Reference) 지원**: 캐릭터/인물의 특징 추출 및 합성 로직 구현 완료.
- [ ] **이미지 전처리 로직**: 서버 부하 감소를 위한 클라이언트측 이미지 리사이징/압축 가이드 수립.
- [ ] **고도화된 에러 핸들링**: 안전 필터 작동 또는 생성 실패 시 사용자에게 전달할 구체적인 가이드 메시지 설계.
- [ ] **성능 최적화**: FastAPI와 Gemini API 간의 통신 지연 시간 최소화.

## 📱 2단계: 모바일 프론트엔드 (React Native & Expo)
*목표: 직관적인 케이크 디자인 및 주문 사용자 경험 구축*

- [ ] **프로젝트 초기 설정**: Expo SDK 기반의 React Native 개발 환경 구축.
- [ ] **대화형 캔버스 UI**: 손가락으로 수정 영역을 칠할 수 있는 마스킹(영역 지정) 컴포넌트 개발.
- [ ] **이미지 관리 시스템**: 갤러리 접근 및 카메라 촬영 연동 (원본 및 참고용 사진).
- [ ] **실시간 시안 뷰어**: 서버 응답(Base64)을 렌더링하고 '수정 전/후' 비교 기능 구현.
- [ ] **주문 채팅 인터페이스**: 자연어 기반 주문 및 정보 자동 추출을 위한 챗봇 UI 구축.
- [ ] **매장 탐색 기능**: 케이크 샵 목록 조회 및 포트폴리오 상세 페이지 개발.

## ☕ 3단계: 메인 비즈니스 백엔드 (Spring Boot)
*목표: 사용자, 주문 관리 및 전체 비즈니스 로직 제어*

- [ ] **DB 스키마 설계**: 유저(구매자/판매자), 매장, 주문, 디자인 시안 관리를 위한 RDBMS 설계.
- [ ] **인증 및 보안**: JWT 기반 인증 및 역할 기반 접근 제어(RBAC) 구현.
- [ ] **주문 라이프사이클 API**: [시안 확정 -> 결제 대기 -> 제작 중 -> 수령 완료] 상태 관리 로직.
- [ ] **AI 엔진 게이트웨이**: FastAPI로 요청을 전달하기 전 유효성 검사 및 로그 기록.
- [ ] **사장님 대시보드 API**: 포트폴리오 관리 및 일일 주문 현황 통계 제공.

## 🤖 4단계: 비즈니스 AI 및 플랫폼 확장
*목표: AI를 활용한 매장 운영 최적화 및 고객 서비스 강화*

- [ ] **주문 정보 자동 추출**: 채팅 내역에서 [맛, 사이즈, 문구, 날짜] 등 핵심 정보 자동 구조화.
- [ ] **리뷰 감성 분석**: 고객 피드백 요약 및 주요 개선 포인트 제안 AI.
- [ ] **데일리 브리핑 엔진**: 사장님을 위한 일일 제작 일정 및 주의사항 자동 보고서 생성.
- [ ] **AI 컨시어지**: 알러지 체크 및 초기 디자인 상담 자동 응대.

## 🚀 5단계: 통합 테스트 및 배포
*목표: 시스템 안정성 확보 및 서비스 런칭*

- [ ] **E2E(End-to-End) 테스트**: [모바일 앱 -> Spring Boot -> FastAPI -> Gemini] 전체 흐름 검증.
- [ ] **클라우드 인프라 구축**: Docker를 활용한 마이크로서비스 배포 (AWS/GCP 등).
- [ ] **모니터링 및 로깅**: 디버깅 및 성능 추적을 위한 중앙 집중식 로깅 시스템 구축.
- [ ] **베타 테스트**: 실제 케이크 매장과 협력하여 피드백 수렴 및 서비스 개선.
