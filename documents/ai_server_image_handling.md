# AI 서버 이미지 처리 방식 가이드 (URL 및 Base64 하이브리드 지원)

안녕하세요! 이 문서는 AI 서버의 이미지 인페인팅(편집) API가 어떻게 이미지를 주고받는지 주니어 개발자분들도 쉽게 이해하실 수 있도록 설명하기 위해 작성되었습니다.

## 1. 왜 로직을 수정했나요?

기존에는 AI 서버로 이미지를 보낼 때 **항상 Base64 형태의 긴 문자열**로 변환해서 보내야 했습니다.
Base64로 변환하면 이미지 용량이 원본보다 약 30% 커집니다. 프론트엔드 ➡️ 백엔드 ➡️ AI 서버로 통신할 때 이렇게 커진 데이터를 계속 주고받으면 서버 메모리(RAM)가 낭비되고 통신 속도도 눈에 띄게 느려지는 문제가 발생합니다.

이 문제를 해결하기 위해, 이미 S3 같은 스토리지에 올라가 있는 이미지는 무겁게 Base64로 만들지 않고 **간단하게 URL 주소만 보내도록** 코드를 개선했습니다!

## 2. 무엇이 바뀌었나요?

### ✅ `InpaintRequest` 데이터 모델 변경 (`main.py`)
기존에는 `image_b64`, `mask_b64`만 있었지만, 이제는 `_url` 필드와 `_b64` 필드를 모두 받을 수 있도록(옵셔널) 변경되었습니다.

```python
class InpaintRequest(BaseModel):
    prompt: str

    # 원본 이미지 (URL 방식과 Base64 방식 모두 지원)
    image_url: str = None  # URL로 보낼 때 사용 (우선 적용됨)
    image_b64: str = None  # 기존처럼 Base64로 보낼 때 사용
    
    # 마스크 이미지, 참고 이미지도 동일한 방식으로 필드가 추가되었습니다.
```

### ✅ 하이브리드 이미지 로더 `load_image()` 추가
요청으로 들어온 데이터가 URL인지 Base64인지 판별하고, 모델이 읽을 수 있는 이미지 객체(PIL Image)로 만들어주는 헬퍼 함수를 추가했습니다.
URL이 넘어오면 `requests` 라이브러리를 통해 AI 서버가 알아서 S3(또는 외부 저장소)에서 이미지를 다운로드합니다.

## 3. 프론트엔드 & 백엔드 실무 사용 꿀팁!

이제 상황에 맞게 가장 편하고 효율적인 방식을 섞어서(하이브리드) 사용하시면 됩니다.

* **👍 추천하는 조합 (원본은 URL, 마스크는 Base64)**
  포트폴리오 원본 이미지는 이미 S3에 있으니 가볍게 URL로 넘겨주고, 유저가 방금 캔버스에 그린 마스크 이미지는 스토리지에 굳이 업로드하지 말고 바로 Base64로 넘겨주세요!
  ```json
  {
    "prompt": "하트 그려줘",
    "image_url": "https://s3.aws.com/.../original.png",
    "mask_b64": "iVBORw0KGgoAAAANSUhEUgAAAAE...",
    "reference_image_url": "https://s3.aws.com/.../reference.png"
  }
  ```

* **전부 다 Base64로 보내도 되나요?**
  네! 하위 호환성을 완벽하게 유지했기 때문에 기존에 사용하시던 `_b64` 필드를 그대로 쓰셔도 잘 동작합니다.

코드를 보시다가 이해가 안 가는 부분이 있다면 언제든 편하게 질문해 주세요! 화이팅입니다! 🚀
